## Contract boundaries

- Bridge HTTP contract source: `packages/bridge/Source/UnrealOpenMcpEditor/Private/Bridge/` (`GET /ping` + `POST /tools/{name}` dispatch with the `{ok,result,error}` envelope shipped; mutating dispatches widen the envelope with a `gate` summary block per [Gate policy](api/bridge-http.md#gate-policy))
- Gate policy source: `packages/bridge/Source/UnrealOpenMcpEditor/Private/Gate/` (`UnrealOpenMcpGatePolicy` wraps every mutating dispatch in `checkpoint → mutate → validate → delta`; `UnrealOpenMcpVerifyGateAdapter` is the bridge→verify glue with the rule-selection + filter surface the meta-tools consult; `UnrealOpenMcpCheckpointStore` mirrors the gate's checkpoint for the meta-tools)
- Gate meta-tools source: `packages/bridge/Source/UnrealOpenMcpEditor/Private/MetaTools/UnrealOpenMcpGateMetaTools.h` (the three read-only tools `validate_edit` / `checkpoint_create` / `delta` — the explicit checkpoint → mutate → delta surface; bypass `GatePolicy.Execute` so they do not recurse)
- apply_fix source: `packages/bridge/Source/UnrealOpenMcpEditor/Private/MetaTools/UnrealOpenMcpApplyFixTool.h` (the fix-application workflow — dry-run preview lists the description / safe flag; non-dry-run applies run through the gate runner with a `FixRollback` snapshot so a corrupting fix is auto-reverted on failure or new errors under `enforce`; v1 ships the single Safe provider `clear_broken_soft_reference`)
- Instance lock + port resolver source: `packages/bridge/Source/UnrealOpenMcpRuntime/Public/Bridge/UnrealOpenMcpInstancePortResolver.h` (deterministic port + lock path), `packages/bridge/Source/UnrealOpenMcpEditor/Private/Bridge/UnrealOpenMcpBridgeInstanceLock.h` (lock file writer)
- SHA-256 source: `packages/bridge/Source/UnrealOpenMcpRuntime/Public/Crypto/UnrealOpenMcpSha256.h` (self-contained FIPS 180-4; byte-for-byte parity with Node `crypto.createHash('sha256')`)
- MCP server routing/registry source: `mcp-server/src/index.ts` (stdio server + tool dispatch), `mcp-server/src/live-client.ts` (live bridge HTTP client; routes `ping` → `GET /ping` and every other tool → `POST /tools/{name}`)
- MCP capabilities surface (local rule/fix catalog + builder): `mcp-server/src/capabilities/` (shipped — `rule-catalog.ts` mirrors the implemented verify rules and fix providers; `build-capabilities.ts` assembles the capabilities payload; `unreal_open_mcp_capabilities` is local-route, no bridge hop)
- MCP tool definitions source: `mcp-server/src/tools/` (`ping` + the actor family shipped: `actor_find`, `actor_create`, `actor_modify`, `object_modify`, `actor_set_parent`, `actor_duplicate`, `actor_destroy`, and the five `actor_component_*` tools; plus the level family shipped: `level_open`, `level_save`, `level_list_loaded`, `level_set_current`, `level_unload_sublevel`, `level_get_data`, `level_create`; plus the gate meta-tools shipped: `validate_edit`, `checkpoint_create`, `delta`; plus `apply_fix`; plus `capabilities` (local-route discovery); plus the asset read family shipped: `asset_find`, `asset_get_data`; plus the Content Browser CRUD family shipped: `asset_create_folder`, `asset_copy`, `asset_move`, `asset_delete`, `asset_refresh`; plus the material family shipped: `material_create`, `material_modify`, `material_get_data`; plus asset import shipped: `asset_import` (absolute host file → `/Game` folder via `IAssetTools::ImportAssetTasks`; supported types depend on installed importers — textures/static meshes/sounds; `replace_existing` / `save` default false); plus the editor family shipped: `editor_application_get_state` / `editor_application_set_state` (P5.1 — PIE get/set: read-only `{ isPlaying, isPaused, isSimulating, editorMap }` snapshot + start/stop/pause/resume driver; start/stop return latent `{ pending:true }` and the agent polls get-state, invalid transitions return `invalid_transition`; set-state mutating, gate Enforce, `paths_hint` required) and `editor_selection_get` / `editor_selection_set` (P5.2 — actor selection get/set: get returns `{ count, actors: IdentityRef[] }`; set replaces by refs or clears with explicit `clear:true`, resolve-before-mutate aborts on a bad ref, empty call refused; set mutating, gate Enforce, `paths_hint` required); plus the console family shipped: `console_get_logs` / `console_clear_logs` / `console_run_command` (P5.3 — bounded GLog ring-buffer collector: get is a filtered read (verbosity min-severity / category / substring + limit) over a session-scoped recent-N window (NOT a full Output Log mirror; cold-start gap + overflow documented), clear empties the buffer (read-only; buffer-local), run-command runs `GEngine->Exec` and returns `{ command, output, handled }` — mutating, gate Enforce, `paths_hint` required, destructive accepted-risk) and the reflection family shipped: `reflection_method_find` / `reflection_method_call` (P5.4 — first-party UFunction reflection, no ReflectorNet: find lists UFunctions with signature/flag descriptors + matched/returned honesty; call invokes by `method` on `target` XOR `class` (CDO) with a safety allow-list (BlueprintCallable / CallInEditor only; CDO path requires static / CallInEditor) — mutating, gate Enforce, `paths_hint` required, same accepted-risk class as run-command); further families TBD)
- MCP resources source: `mcp-server/src/resources/` (when shipped)
- E2E smoke sources: `mcp-server/src/integration.test.ts` (in-process MCP client↔server over in-memory transport — ping, `actor_find`, and `asset_find` round-trips), `mcp-server/scripts/p1-parity-smoke.mjs` / `p2-parity-smoke.mjs` / `p4-parity-smoke.mjs` (`npm run smoke:p1` / `smoke:p2` / `smoke:p4`; scripted stdio smoke against the built `dist/index.js`; healthy / bridge-down / tool-error cases). See [E2E smoke verification](architecture.md#e2e-smoke-verification).

## Contract documentation guidance

- Prefer documenting behavior and payload shapes over implementation details.
- Call out breaking changes explicitly.
- Keep examples minimal and representative.
- Unreal content paths use `/Game/...`; document live/offline/local/batch routes per tool.

Update this index when:

- contract ownership moves to new modules,
- endpoint or resource domains are reorganized.
